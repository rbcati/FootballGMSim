perf: optimize playoff bracket filtering

💡 What: Mapped the 'winners' array of team IDs into objects with 'teamId', 'seed', and 'conf' in a single pass before evaluating conference loops.
🎯 Why: Previously, filtering and mapping occurred inside loops for each conference (and each playoff round week), which caused duplicate lookups via 'getConf' and 'getSeed' and repeated array traversal. This issue was especially prevalent in 'currentWeek === 19' and 'currentWeek === 20'.
📊 Measured Improvement: A microbenchmark was run against an expected 6-game wild-card winner array simulating the logic inside the loop. The baseline took ~721ms over 2,000,000 iterations. Pre-mapping the results down to objects and then performing a simple filter reduced execution time to ~591ms (an ~18% improvement for this segment). This ensures clean linear complexity scaling relative to the number of remaining teams per round.
